🧠 Catcher Treatment Engine — 엔진 매뉴얼

Boston Neuromind · 증상 무관 진단·훈련·진화 엔진의 전체 구조 문서. 코드는 공통, 데이터(rules)만 교체해 증상을 복제한다.

① 설계 철학

"AI 적용 안 되면 nothing." 진짜 AI = LLM 호출(검색)이 아니라 판단 메커니즘 + 스스로 자기발전하는 루프. 데이터가 쌓일수록 정밀해지는 구조여야 카피 불가 IP가 된다.

핵심 원칙 4가지

원칙의미
엔진 차원ADHD는 첫 콘텐츠일 뿐. 코드는 증상 무관, rules 데이터만 갈아끼워 10+ 증상 복제.
1세대 → 2세대논문 기반 합성으로 지금 작동(부트스트랩) → 실측 쌓이면 표상/정책 학습으로 진화.
측정 = 훈련게임 1판이 훈련이자 측정기(behavioral probe). 묻지 않고 재게 한다.
판단 전시엔진의 내부 추론(z지문·교차판별·적응)을 화면에 꺼내면, 같은 데이터가 "판단하는 AI"로 보인다.
정직한 경계선: 표상·정책 학습은 실측 N이 쌓여야 켜진다. N=0이면 못 돈다. 그래서 1세대(논문 합성)로 첫 데이터를 걷고, behavioral probe가 그 데이터를 누적한다.

② 전체 흐름 (사용자 여정)

입력(나이·BPS) → 설문(진단) → QEEG 측정 → 알고리즘 리포트 → AI 심층분석 → 뉴로피드백 → HRV·Temp·GSR → AI 치료훈련(게임)

AI 치료훈련 탭 내부 흐름 (핵심 엔진)

1
연령 선택 — 아동(6–12, 보호자)/성인(12+, 본인). 문항·표현이 갈린다.
2
3층 설문 — L1 진단(DSM-5) → L2 매칭(5조절축) → L3 참여스타일(SDT). 각 설문 전 "무엇을 재는지" 설명.
3
QEEG 옵션 — 측정하면 theta/beta로 축 보정, Skip 가능.
4
매칭 파이프라인 — 3레이어 곱하기 → 주요 타겟 축 + 강도 + 게임화 스타일.
5
보고서 — 개별 특징 + 왜 이 훈련인지 (표·바·문장·근거).
6
처방 — 분/일·일/주·주 (조절 가능 select). 총 세션 재계산.
7
게임 선택·실행 — 추천 3 + 전체 리스트박스. 선택 시 WHY 설명 + 인라인 시작.
8
게임 후 4단 전시 — 행동측정 판단 → 또래 z → 교차판별 → 적응 판정.
9
진전 모니터링 — baseline 대비 향상, 목표 달성도, 세션 추이 (영속 저장).

③ 3층 AI 구조 (경쟁력의 심장)

층위AI 능력데이터 필요현재 상태
L1 교차판별설문+QEEG+행동을 융합해 사람이 못 잇는 판별중간1세대 가동 (norm 기반)
L2 적응처방반응 결과로 난이도·게임 순서 학습많음1세대 가동 (곡선 기반)
L3 해석생성엔진 근거를 LLM에 주입해 개인 해석 문장0API 대기 (자리 준비)

L1 — CrossEngine (교차 판별)

게임 측정값 → 문헌 norm 대비 z지문 → 설문 축과 교차 판별. 예: "설문은 주의문제를 가리키나 행동 z지문은 충동(commission +2.1σ)이 더 이탈 → 근본은 억제." 규칙표가 절대 못 하는 것.

교차 판별 3규칙

패턴의미
설문↑ 행동정상상황 특이적이거나 주관적 부담 — 특정 맥락 확인
설문정상 행동↑본인이 인식 못 한 조절 어려움 — 객관 신호
설문↑ 행동↑ (수렴)두 독립 소스 일치 → 훈련 필요 신뢰 가능

L2 — AdaptEngine (적응 처방)

세션 추이를 문헌 기대곡선(세션당 ~1.5% 향상)과 비교 → 판정: 🚀 빠름→난이도↑✅ 정상→유지🔄 정체→게임전환🐢 느림→난이도↓🏆 달성→다음축

L3 — LLM 해석 (API 대기)

엔진의 z지문·교차판별·차등을 LLM 프롬프트에 주입 → "당신의 억제율은 정상보다 1.8σ 낮고, RT변동성은 정상이라 순수 주의문제와 감별됩니다…" 검색형 챗봇은 근거 데이터가 없어 못 한다. API 켜지면 같은 데이터가 AI 해석으로 격상.

                ├─→ 세션 저장(영속) ─→ 진전·모니터링

⑦ QEEG · NF 통합 (설계)

QEEG → 축 보정

theta/beta @ Cz 측정 → qeeg_adjust 규칙 → 주의·각성 축 z 보정. 근거: FDA NEBA(2013, 보조지표), Monastra, Snyder(2006 +32% theta). 진단 단독 아님, 보조.

NF 처방 (L2 정책학습의 대상)

전통: 규칙표(이 대역 높으면 이 프로토콜). AI 목표: QEEG 상태 → 어느 NF 걸면 다음 측정이 좋아지는지 결과로부터 정책 학습 (adaptive-core Q함수). 게임 적응(L2)과 같은 엔진 구조 — NF도 반응곡선 비교 → 프로토콜 적응으로 이식 가능.
미배선: HRV·Temp·GSR 탭(뉴로피드백 다음), NF 적응 엔진은 로드맵. 게임 L2 AdaptEngine을 NF에 재사용하는 것이 설계 방향.

⑧ 진화 루프 (1세대 → 2세대)

1세대 (현재)
논문 기반 합성 norm으로 z지문·교차판별·적응이 지금 작동. 데이터 0에서 부트스트랩. behavioral probe가 실측을 걷는다.
2세대 (진화 후)
실측 N 쌓이면 norm 자동 블렌딩(문헌→실측). 표상학습(3소스 융합), 정책학습(처방 최적화)으로 격상.

블렌딩 공식

w_real = n_real / (n_real + K)   # K=50
blended = 문헌 × (1 - w) + 실측 × w   # 캡 95% (문헌 최소 5% 유지)

n=0   → 문헌 100%  (1세대)
n=50  → 실측 50%
n=500 → 실측 91%   (2세대)
화면에 정직하게 표시: "기준: 문헌 norm (실측 N건 누적 중 — 쌓일수록 정밀해집니다)" → 사용자가 진화를 체감.

⑨ 증상 복제 절차 (예: OCD)

1
axes — OCD 조절축 재정의 (예: 강박사고·강박행동·불확실성 인내…)
2
surveys — Y-BOCS 등 OCD 문헌으로 3층 설문 교체
3
games — 5엔진 재사용, 파라미터만 OCD 훈련 목표로 (억제·불확실성 노출 등)
4
norms — OCD 문헌 효과크기로 지표 norm 교체
5
adaptation — OCD 훈련 반응곡선
6
코드는 0줄 수정. 엔진 모듈 전부 재사용. 도메인만 새로.
Genius 저작 필수: 설문 문항·축 정의·norm 수치·반응곡선은 임상 저작. 코드가 아니라 데이터가 IP.
